# 第三章实战示例：从函数开始写测试

这一章用“密码校验器”来练习最适合单元测试的起点：**纯函数**。

---

## 为什么从函数开始

纯函数通常比类、接口、I/O 更容易测试，因为它的输入输出关系清晰：

- 输入：密码字符串
- 输出：是否通过 + 错误列表

当你刚开始学测试时，先从这种场景入手最容易建立信心。

---

## 需求

- 长度至少 8 位
- 必须包含字母
- 必须包含数字
- 不能包含空格

---

## 推荐的 TDD 顺序

### 1. 先写最简单的通过场景

```python
assert validate_password("abc12345") == (True, [])
```

### 2. 再补边界条件

```python
assert validate_password("ab12") == (False, ["密码长度至少 8 位"])
```

### 3. 再补缺少字母、缺少数字、包含空格等场景

```python
assert validate_password("12345678") == (False, ["密码必须包含字母"])
```

---

## 代码位置

- 实现：`src/tdd_learning_python/chapter3.py`
- 测试：`tests/test_chapter3.py`

---

## 你应该观察什么

- 通过场景和失败场景都要测
- 一次只加一条规则，测试更容易读懂
- 错误列表比单纯的 True / False 更适合教学，因为它更容易看出失败原因

这个例子重点不是写出“最复杂”的校验器，而是学会：

> **先从最容易验证的纯函数开始，再把规则拆细，最后用测试驱动实现。**
