# 练习 06：测试架构

## 目标

学习如何组织较大的测试集，让测试既好写又好维护。

## 需求示例

- 用 fixture 准备共用数据
- 用参数化测试覆盖一组回归场景
- 用测试数据工厂生成可读性更好的测试数据

## 建议

- 先从重复很多的测试数据开始整理
- 让测试意图比实现细节更重要
- 优先保证可读性，再考虑抽象复用

- 阅读 `docs/chapter-6-cli-e2e.md`

- 什么时候该用 fixture，什么时候不该用？
- 测试工厂会不会过度抽象？
